웹서비스를 하려면 하기 위한 알고리즘 로직을 프론트엔드로 돌아가야 함

1. 웹 서비스에서 데이터 포맷 선택 기준
웹 서비스를 개발할 때, 알고리즘 로직은 프론트엔드에서 실행되며,
프론트엔드는 JavaScript를 사용해야 함.
데이터 교환 시 JSON을 사용하는 것이 좋음.

2. 데이터 포맷 선택 기준
JSON
- 키-값 쌍 기반의 경량 데이터 형식
- 웹과 모바일 API에서 표준으로 사용
- JavaScript와 호환성이 뛰어남	웹 서비스에서 서버-프론트엔드 간 데이터 교환

XML
- 태그 기반의 구조화된 데이터 형식
- 데이터가 길거나 세부적인 정보 포함 가능
- 문서 저장, 설정 파일에 적합	긴 문서나 복잡한 데이터 구조를 저장할 때 적합

CSV
- 쉼표(,)로 구분된 데이터
- 단순한 테이블 형태의 데이터 저장 가능
- 사람이 읽고, 엑셀 등에서 쉽게 활용 가능	표(테이블) 형태의 데이터를 정리할 때 적합

3. 결론
- 웹 서비스에서는 JSON을 사용하는 것이 가장 효율적
- 내용이 길고, 세부적인 구조가 필요하다면 XML
- 데이터를 테이블 형식으로 정리하려면 CSV
- 목적에 맞게 적절한 데이터 포맷을 선택해야 함
